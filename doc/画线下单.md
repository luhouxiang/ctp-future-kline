结合以下的资料，实现我们模拟交易的画线下单：
我会把“划线下单”理解为：在K线/分时图上画价格线或趋势线，行情触碰/突破后自动在模拟账户下单。下面会从架构、触发逻辑、模拟交易接入、风控和测试路径来展开。

Thought for 58s

可以按“本地画线触发 + CTP/SimNow 模拟报单”来做。核心不是让交易所识别你画的线，而是你的客户端持续接收行情，判断价格是否触线/穿线，然后通过模拟交易接口发出普通报单或条件报单。

1. 推荐总体架构
   前端图表层
   负责K线/分时图展示、画水平线/趋势线、设置下单参数。

策略触发层
把用户画的线转换成规则，例如：

合约：rb2405
线型：水平线 / 趋势线
触发方式：上穿 / 下破 / 触碰
确认方式：Tick触发 / K线收盘确认
下单方向：买开 / 卖开 / 买平 / 卖平
数量：1手
价格：限价 + N跳 / 对手价 / 最新价偏移
有效性：触发一次后失效

行情层
通过 CTP 行情接口登录并订阅合约行情。CTP 客户端开发流程通常是初始化、连接、登录、订阅行情，然后通过 OnRtnDepthMarketData 持续收到行情推送；CTP 前置地址格式类似 tcp://127.0.0.1:17001。

交易层
触发后调用 CTP 交易接口 ReqOrderInsert 报单，用 OnRtnOrder 跟踪报单状态，用 OnRtnTrade 确认成交。CTP 文档也建议以成交回报作为成交标志。

模拟环境
可先接 SimNow。上期所页面介绍，SimNow 是上期技术为投资者提供的期货、期权模拟交易平台，可仿真交易、结算规则，并引入实盘价格信息用于仿真。

2. 画线触发逻辑
   水平线
   水平线最简单：

line_price = 3650

上穿触发：
prev_price < line_price and last_price >= line_price

下破触发：
prev_price > line_price and last_price <= line_price

报单方式选择
方案A：本地触发，发普通限价单
这是最稳妥的实现方式。

触发后由客户端生成普通报单：

ContingentCondition = Immediately
OrderPriceType = LimitPrice
LimitPrice = 触发价 ± N跳

CTP 报单字段中，普通限价单通常是“立即触发 + 限价类型”；限价单需要指定价格，否则会报错。

优点：
实现简单，逻辑完全可控，适合划线下单、趋势线、区间突破、K线确认等复杂条件。

报单后要维护订单索引。CTP 中一笔报单可用 FrontID + SessionID + OrderRef 标识，OrderRef 可从登录后返回的 MaxOrderRef 递增维护；也可用交易所返回的 ExchangeID + OrderSysID 跟踪。

6. 必须做的风控
   划线下单看起来像手工交易，但技术上属于自动触发报单，必须加风控：

一线一单：默认触发一次后禁用，防止反复穿线重复下单。
最大委托频率：例如每秒最多1笔，每分钟最多N笔。
价格保护：限价不能偏离最新价太多，例如超过10跳拒绝。
交易时段校验：集合竞价撮合阶段、交易小节休息时段下单可能被拒。国联期货交易时间说明中也提示，集合竞价撮合时间和小节休息时间下单会被交易系统拒绝并视为废单。
断线保护：行情断线、交易断线、心跳异常时暂停触发。
撤单保护：触发后若N秒未成交，可自动撤单或提示用户。
全局急停：界面上必须有“一键停止所有划线触发”的按钮。
总体上，划线下单的最佳实现方式是：图形界面只负责画线和参数配置，真正的触发、风控、报单、回报处理放在独立交易引擎中；模拟交易优先接 SimNow，先用普通限价单完成闭环，再考虑条件单和更复杂的订单类型。




已实现“模拟交易画线下单”闭环：

* 后端新增内存态画线下单引擎：**line_order.go**
  * 支持水平线、趋势线
  * 支持 **touch** / **cross_up** / **cross_down**
  * 触发后调用模拟交易 **SubmitOrder**
  * 一线一单，触发后规则失效
  * 限价偏移跳数、最大偏离 10 跳保护
  * 委托频率保护：每秒 1 笔、每分钟 10 笔
  * 全部停止接口
* Web API 接入：**server.go**
  * **GET /api/trade/line-orders**
  * **POST /api/trade/line-orders**
  * **POST /api/trade/line-orders/{id}/disable**
  * **POST /api/trade/line-orders/stop-all**
* 交易风控允许 **line_order** 来源：**risk.go**
* 前端对象树接入：**TradingTerminal.vue**、**WatchlistPanel.vue**
  * 对水平线/趋势线悬停显示“画线下单”
  * 已启用规则可停止
  * 对象树顶部有“全部停止”急停
  * 参数用 prompt 输入：触发方式、买卖方向、开平、手数、限价偏移跳数
