# 半自动 MA20 反抽做空策略

## Summary

- 基于现有 `internal/strategy` 架构实现一个 Python 策略：`ma20.pullback_short`。
- Go 继续负责行情/tick/K线推送、策略实例管理、信号持久化、模拟执行与审计。
- Python 只维护策略状态机并输出交易意图，不直接下单。
- 第一版行为固定为：手动启动实例、tick 最新价跌破触发、模拟自动执行、触发一次后停止发信号。

## Key Changes

- Python 策略服务新增 `ma20.pullback_short` 定义：
  - 默认参数：`ma_period=20`、`max_wait_bars=6`。
  - 输出：`target_position=-1`、`confidence`、`reason`、`metrics`。
  - `metrics` 至少包含：`signal=SHORT`、`state`、`touch_open`、`touch_high`、`touch_ma20`、`touch_time`、`trigger_price`。
  - 不计算 `stop_loss`，只输出 `touch_high` 作为止损参考。
- Python 按 `instance_id + symbol` 维护独立状态：
  - `WAIT_BREAK_BELOW_MA20`
  - `BROKEN_BELOW_MA20`
  - `WAIT_BREAK_TOUCH_OPEN`
  - `DONE`
- Go 侧保持现有策略信号接口：
  - `SignalDecision.TargetPosition=-1` 表示固定 1 手空头目标仓位。
  - 现有 `ExecutionEngine` 继续模拟更新仓位并写 `order_audit_logs`。
  - 不接入实盘自动下单，不修改 `trade.block_strategy_live_order` 的默认阻断语义。

## Strategy Rules

- MA20 使用 Python 内部根据已完成 K 线 `close` 计算的 SMA20。
- 跌破判定：
  - 上一根 K：`prev_close >= prev_ma20`
  - 当前 K：`close < ma20`
  - 满足后进入 `BROKEN_BELOW_MA20`。
- 反抽触碰：
  - 后续 K 线 `high >= ma20`
  - 该 K 线记录为触碰 K：
    - `touch_open = open`
    - `touch_high = high`
    - `touch_ma20 = ma20`
    - `touch_time = data_time`
  - 进入 `WAIT_BREAK_TOUCH_OPEN`，该触碰 K 计为等待窗口第 1 根。
- SHORT 触发：
  - 在 `WAIT_BREAK_TOUCH_OPEN` 中，任一 tick `last_price < touch_open`。
  - 输出一次 `SHORT`，随后实例状态进入 `DONE`，不再重复发信号。
- 失败重置：
  - 触碰 K 之后，若任一后续完整 K 线 `open > ma20 && close > ma20`，视为站稳 20MA，重置为等待新的完整跌破。
  - 若计数到第 6 根 K 线仍未出现 tick 跌破 `touch_open`，本轮失败。
  - 失败后重新等待“完整跌破 20MA”：`open < ma20 && close < ma20`，再进入下一轮 `BROKEN_BELOW_MA20`。

## Interfaces And Storage

- 不新增数据库表；复用：
  - `strategy_definitions`
  - `strategy_instances`
  - `strategy_signals`
  - `order_audit_logs`
- 不新增 HTTP API；复用：
  - `POST /api/strategy/instances`
  - `POST /api/strategy/instances/{id}/start`
  - `GET /api/strategy/signals`
  - `GET /api/orders/audit`
- Python 服务内部需要把示例策略逻辑拆成可注册策略类，避免 `sample.momentum` 和新策略混在同一个 `_decision` 分支里。

## Test Plan

- Python 单元测试：
  - 不足 20 根 K 线不产生状态变化。
  - 正确识别 `prev_close >= prev_ma20 && close < ma20`。
  - 触碰 K 记录 `open/high/ma/time`。
  - tick `last_price < touch_open` 只产生一次 SHORT。
  - 后续 K 线 `open > ma20 && close > ma20` 后失败重置。
  - 第 6 根仍未跌破 `touch_open` 后失败重置。

## Assumptions

- 第一版只做空头形态，不实现多头镜像。
- 第一版固定 1 手目标空头仓位，即 `target_position=-1`。
- 第一版不输出 `stop_loss`，只输出 `touch_high` 作为后续人工或 UI 参考。
- 第一版触发一次后停止该策略实例的信号输出，避免重复开仓。
- 第一版不做前端新页面；通过现有策略实例 API 和信号/审计列表验证闭环。
