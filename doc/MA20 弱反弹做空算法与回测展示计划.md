# MA20 弱反弹做空算法与回测展示计划

## Summary

- 新增策略 `ma20.weak_pullback_short`，保留现有 `ma20.pullback_short` 作为基准算法。
- 数据源固定使用 `market_realtime` 逻辑库，即 `future_market_realtime`，测试 5m 数据表：
  `future_kline_l9_mm_sr`、`future_kline_l9_mm_ss`、`future_kline_l9_mm_sp`、`future_kline_l9_mm_y`、`future_kline_l9_mm_ao`。
- 成败定义采用已确认默认：信号后 24 根 5m K 内，先跌破 `entry - 1.0*ATR14` 为成功；先上涨到 `entry + 0.8*ATR14` 为失败；同一根 K 同时触发上下阈值按失败统计。
- 展示做全量：右侧策略步骤、K 线图表标记、批量回测报告。

## Key Changes

- 算法流程：

  - 从每次 `prev_close >= prev_ma20 && close < ma20` 创建一次 attempt。
  - 先做趋势过滤：`ma20 > ma60 > ma120 && ma60_slope > 0 && ma120_slope > 0 && close > ma60` 时，若未破最近 swing low，直接标记为上涨回撤/停顿，不做空。
  - 再做结构过滤：必须 `close < last_swing_low`，否则最多等待 3 根 K；快速收回 MA20 且未破结构则失败。
  - 只允许 `BEARISH_REVERSAL_CANDIDATE` 或 `WEAK_BEARISH_CANDIDATE` 进入反抽逻辑。
  - 反抽阶段等待 `high >= ma20`，记录触碰 K 与反抽区间低点。
  - 触发阶段从下一根 K 开始，`low <= reaction_low` 形成 SHORT，`entry_price = reaction_low`。
  - 若重新站上 MA20、等待超时或 bullish_pause_score 高于 bearish_failure_score，则 attempt 失败并继续寻找下一个起点。
- 回测比较三套算法：

  - `baseline`: 旧 MA20 跌破、触碰、跌破触碰 K 开盘价逻辑。
  - `hard_filter`: 趋势过滤 + 结构破坏 + 弱势反抽失败。
  - `score_filter`: 在 hard_filter 基础上比较 `bullish_pause_score` 与 `bearish_failure_score`，偏多停顿分更高则过滤。
  - 报告同时给出 `attempt_success_rate = success / attempts_started`、`signal_success_rate = success / signals`、`signal_formation_rate = signals / attempts_started`。
- 接口与工具：

  - Python 策略服务新增 `ma20.weak_pullback_short`，实时/回放时输出更细的 trace。
  - Go 侧新增纯 Go 回测引擎，读取 MySQL 5m K 线并输出 JSON 报告，避免把几十万根 K 线通过 gRPC 发给 Python。
  - 新增 CLI：`cmd/ma20_weak_backtest`，默认跑上述 5 张表和三套算法。
  - 扩展 `/api/strategy/backtests` 结果：summary 包含分算法、分表统计；新增读取详细 result 的接口用于前端报告展示。
  - 前端策略面板更新步骤列表，并把 strategy traces 传入 K 线图，绘制起点、过滤、结构破坏、触碰、SHORT、成功/失败标记。

## Test Plan

- Python 单元测试：

  - 强上涨趋势且未破前低时过滤做空。
  - 跌破 MA20 但快速收回时归类为 `BULLISH_PAUSE`。
  - 跌破 MA20 + 跌破 swing low + 反抽站不回去时形成 SHORT。
  - 触碰 K 当根不允许同时触发入场，必须从下一根 K 判断。
  - replay/live 状态隔离仍成立。
- Go 单元测试：

  - MA20/MA60/MA120、ATR14、swing low、斜率计算正确。
  - 三种算法在固定样本上的 attempt、signal、success/failure 计数正确。
  - 同一根 K 同时碰到止盈/不利阈值按失败统计。
  - MySQL 表缺失、空表、MA 样本不足时返回清晰错误或跳过结果。
- 集成验证：

  - 按 AGENTS.md 先设置：

  - 跑 `go test ./internal/strategy ./internal/web`。

  - 前端跑现有 web 测试，并手动用 K 线回放确认右侧步骤、审计流、图表标记同步更新。

## Assumptions

- 使用 24 根 5m K 作为信号成败观察窗口。
- 成功/失败阈值使用 ATR 动态点数：`profit = 1.0*ATR14`，`adverse = 0.8*ATR14`。
- 价格触发只使用已完成 5m K 线，不使用 tick；入场价按触发阈值 `reaction_low` 计算。
- 回测报告统计 filtered attempts，但 filtered attempts 不计入 signals。
- `future_market_realtime` 中已确认目标表存在，5m 数据范围大致覆盖 2021/2023 至 2026-04-30。
